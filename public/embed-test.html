Vaani widget — embed test

Two ways to embed the widget on any host page.

No JS required

A — Simple <iframe> tag

Drop this <iframe> anywhere on your page. The iframe is fixed at bottom-right via CSS on the host. You must include allow="microphone; autoplay". 

<iframe src="/"
  allow="microphone; autoplay"
  title="Vaani Voice Assistant"
  style="
    position: fixed;
    bottom: 0;
    right: 0;
    border: 0;
    width: 420px;
    height: 750px;
    z-index: 2147483647;
    background: transparent;
  "
></iframe>

This page uses the simple iframe below ↓

Positioned inline (demo). On a real host page you'd use position:fixed;bottom:0;right:0.

Recommended — dynamic sizing

B — Script embed

Include one <script> tag. It creates a fixed-position iframe at bottom-right and dynamically resizes it as the widget opens/closes. The widget appears as a floating FAB when idle and expands to a full chat panel when active. 

<script src="/embed.js"></script>

To point at a different deployment, set data-src: 

<script src="/embed.js" data-src="https://your-domain.com"></script>

The script embed is active below ⇩

(Look at the bottom-right corner of your browser window.)

Important

Requirements

  • Microphone access — the iframe needs allow="microphone". Without it the browser denies mic access inside the frame.
  • Autoplay — add autoplay to the allow list so audio can play without user gesture.
  • CORS — your token endpoint must include Access-Control-Allow-Origin: * (or your host origin) since the widget fetches it cross-origin.
  • sandbox — if your host page uses a restrictive sandbox attribute, include allow-scripts allow-same-origin allow-popups.
